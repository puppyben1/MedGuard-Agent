# Neo4j Import Runbook

This runbook describes how to import the offline SIDER/MedDRA graph artifacts generated by MedGuard-Agent. It does not download or fabricate graph data.

## Inputs

Build the local SIDER/MedDRA artifacts first:

```powershell
.\.venv\Scripts\python.exe scripts\build_sider_meddra_index.py --input data\incoming\adr_data.zip --output data\processed\sider_meddra
```

Expected generated files:

```text
data/processed/sider_meddra/neo4j/drugs.csv
data/processed/sider_meddra/neo4j/meddra_terms.csv
data/processed/sider_meddra/neo4j/drug_side_effect_edges.csv
data/processed/sider_meddra/neo4j/meddra_normalization_edges.csv
data/processed/sider_meddra/neo4j/import.cypher
```

## Neo4j Desktop / Server Import

Copy the generated CSV files into the Neo4j import directory configured for your database. Then run the generated Cypher script:

```cypher
:source import.cypher
```

If your Neo4j browser cannot use `:source`, paste the script contents in batches.

## Runtime Configuration

Open the MedGuard-Agent system configuration page and set:

```text
Neo4j URI
Neo4j username
Neo4j password
Neo4j database
```

The live graph endpoints do not fall back to demo data. If Neo4j is not configured or cannot connect, the API returns a diagnostic error.

## Verification

Use:

```text
GET  /api/graph/status
POST /api/graph/drug-side-effects
POST /api/graph/expand
```

Recommended smoke query:

```cypher
MATCH (d:Drug)-[r:HAS_SIDE_EFFECT]->(s:SideEffect)
RETURN d, r, s
LIMIT 25
```

## Data Boundary

The imported graph represents offline SIDER/MedDRA associations. It is not DrugBank, PubMed, FAERS, or patient-level clinical evidence unless those sources are explicitly imported and labeled separately.
